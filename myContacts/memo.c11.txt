폼 : 웹 문서에서 사용자 입력을 받을 수 있는 형태를 가리키는 용어로 로그인폼이나 주문서폼등이 이에 해당함 
     내용을 입력하는 필드도 다양하고 버튼을 클릭해서 입력한 내용을 서버로 전송하는 기능도 있음 

애플리케이션에서 로그인하거나 물건을 구입할 때 사용자가 제공하는 정보는 모두 폼을 통해 서버로 보내짐

사용자 입력을 받는 폼은 밷엔드 개발자가 직접 작성할 수도 있고 프론트앤드 개발자가 만든 HTML문서를 받아서 백엔드에 연결할 수도 있음 
프론트앤드에서 폼을 다룰 때는 class나 id속성만 있어도 되지만 서버에서 폼 필드의 값을 인식할 때는 name속성을 사용하므로 없다면 반드시 추가해야 함 

폼을 통해 정보를 서버로 보낼 때 사용할 수 있는 요청 방식은 GET과 POST뿐 
GET 방식에서는 주소 URL에 정보를 함께 넣어서 보내고, POST방식에서는 요청본문 즉 req.body에 정보를 담아서 보냄 

PUT, DELETE요청은 AJAX기법을 사용하거나 method-override라는 미들웨어를 사용 
AJAX란 HTTP요청을 만들어서 요청 방식과 넘겨줄 정보 등을 지정해 주는 비동기 통신 방법으로 요청방식과 요청경로를 한꺼번에 넘겨주는 방식을 사용 
method-override미들웨어는 요청방식을 덮어씌우는 방법으로 처리
폼에서 POST방식으로 전송하면 그것을 중간에 받아서 PUT 방식으로 바꿔 서버로 전달하는 식 

인터넷을 통해 서버로 전송할 수 있는 문자는 아스키문자뿐

URL인코딩 : 한글로 작성한 정보를 서버로 전송한다면 한글을 아스키 문자 형태로 바꾸어서 전송하는 것 ( 특수기호를 사용할 때도 인코딩이 필요할 수 있음 )

폼에서 자료를 서버로 전송할 때도 인코딩된 URL형식을 사용함 

파싱 : 이미 인코딩되어 있는 자료를 가져와서 사용하기위해 내용을 풀어주는 과정으로 서버로 전송하는 자료를 애플리케이션에서 사용해야 할 떄 사용 

urlencoded 함수 : 익스프레스에서 인코딩된 URL을 파싱해 주는 미들웨어 함수로 express.urlencoded함수로 파싱된 자료는 req.body에 저장됨 
                  extend옵션ㅇ르 사용해 어떤 모듈을 통해 파싱할지 지정
                    - extend: false => 간단한 문자열이나 단순 배열을 파싱
                    - extend: true => 복잡한 객체나 배열까지 파싱
